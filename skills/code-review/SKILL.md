# 代码审查

对指定的代码文件进行系统化审查。

## 审查维度

1. 功能正确性: 逻辑是否有 bug，边界情况是否处理
2. 代码质量: 命名是否清晰，是否有重复代码，职责是否单一
3. 安全性: 是否存在注入、硬编码密钥等安全漏洞
4. 错误处理: 异常场景是否有妥善处理

## 工作流程

1. 使用 read 工具读取目标文件内容
2. 按照上述四个维度逐一审查
3. 对于每个发现的问题，给出:
   - 问题级别 (严重/警告/建议)
   - 所在行号
   - 问题描述
   - 修复建议
4. 输出审查报告摘要
